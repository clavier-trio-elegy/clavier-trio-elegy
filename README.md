# Клавир-трио «Элегия» — статический сайт для GitHub Pages

Этот проект — **быстрый адаптивный сайт без движков и виджетов**: чистый HTML/CSS/JS, один `index.html`.

## Структура
- `index.html` — вся страница (якоря: #about, #concerts, #programs, #media, #contacts)
- `assets/css/styles.css` — стили
- `assets/js/main.js` — минимальный JS (меню, год, поиск по репертуару)
- `assets/img/` — иконки/обложки (SVG)
- `assets/audio/` — (создайте папку и добавьте mp3)

## Как опубликовать на GitHub Pages
1. Создайте репозиторий, например: `elegy-site`
2. Загрузите файлы в корень репозитория
3. GitHub → **Settings** → **Pages**
4. Source: **Deploy from a branch**
5. Branch: `main` / folder: `/root`
6. Сохраните — сайт станет доступен по адресу `https://<ваш-логин>.github.io/<repo>/`

## Что поменять в первую очередь
- Контакты в секции `#contacts`
- Ссылки на видео в секции `#media`
- JSON-LD в `<head>`: поле `"url"` и `"sameAs"`
- Афишу в секции `#concerts`
- Репертуар в секции `#programs`

## Быстрые советы по скорости
- Фото: используйте `.webp` (ширина 1600px обычно достаточно)
- Старайтесь держать общий вес страницы < 1 МБ
- Не подключайте тяжёлые виджеты — лучше ссылки или «lite-embed» (по запросу добавлю)

Удачи!
